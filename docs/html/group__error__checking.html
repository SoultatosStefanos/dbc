Each DBC assertion comes in three variants.

Critical variant
Used for critical assumptions that must always be verified. Most applications do not benefit from this variant. E.g INVARIANT_CRITICAL, REQUIRE_CRITICAL, ENSURE_CRITICAL.
Regular variant
Most assertions fall in this category. Regular assumptions, whose complexity is constant should be verified like this. E.g INVARIANT, REQUIRE, ENSURE.
Safe variant
This variant should be used to assert for complex assumptions, that can slow down an application. Typically useful for inline functions and checks of linear, or greater, complexities. E.g INVARIANT_SAFE, REQUIRE_SAFE, ENSURE_SAFE.

Additionally each DBC assertion is overloaded, in order to provide a developer friendly error message.
